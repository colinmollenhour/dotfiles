---
name: colin-review-merits
description: Critique the merits of a merge or pull request, not the typical code review.
---

Review the merge or pull request in question, but this is not a code review! I want to focus on the merits of the task itself
and the implementation decisions. ONLY look at the code if needed to verify your assumptions. DO NOT inspect previous code review
comments, they are mostly implementation bug fixes and most likely not important for this type of review. DO fetch extra context
from the issue tracker if there is an issue referenced.
